Objectifs d'apprentissage

  • Monter la pile CAIPE complète avec CNOE IDP Builder (kind, Backstage, ArgoCD, Gitea).
  • Comprendre l'architecture superviseur + sous-agents de CAIPE.
  • Inspecter un template de scaffolder Backstage de dev box.
  • Déclencher un flux self-service avec un seul prompt au superviseur.
  • Tracer la requête à travers Backstage puis la réconciliation ArgoCD.
  • Vérifier la notification sortante via une fixture Slack locale.
  • Lire le log append-only de la fixture pour voir le payload JSON exact.
  • Démanteler proprement le cluster et les ressources du lab.

Environnement virtuel

Sandbox
dev@dev-box-flow: ~
# 1 · Monter la pile via IDP Builder
$ idpbuilder create --use-path-routing
[ok] kind cluster · backstage · argocd · gitea · caipe

# 2 · Vérifier les composants CAIPE
$ kubectl get pods -n caipe-system
supervisor · backstage-agent · argocd-agent · slack-agent  Running

# 3 · Envoyer le prompt self-service au superviseur
$ caipe prompt "Create a dev box for me named alpha"
routing -> backstage -> argocd -> slack

# 4 · Inspecter le payload envoyé a Slack
$ curl -s localhost:9006/log | tail -1
{"channel":"#dev-boxes","text":"Dev box alpha is ready"}

Déroulé du laboratoire

  1. Monter la pile CAIPE

    Lancez idpbuilder create et vérifiez que Backstage, ArgoCD, Gitea et le superviseur CAIPE sont en cours d'exécution.

  2. Explorer le template

    Inspectez le template.yaml du scaffolder Backstage de dev box : paramètres, étapes et action de publication.

  3. Déclencher le flux

    Envoyez « Create a dev box for me named alpha » au superviseur et suivez le routage vers les sous-agents Backstage et ArgoCD.

  4. Observer via Slack

    Interrogez la fixture Slack (port 9006) et lisez le payload chat.postMessage dans le log append-only.

  5. Démanteler

    Exécutez les scripts de nettoyage et idpbuilder delete pour libérer le cluster kind.
