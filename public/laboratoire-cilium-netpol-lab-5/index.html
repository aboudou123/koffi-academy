Instructions

Observer les flux avec Hubble

Hubble est l'observabilité de Cilium. Vous allez vérifier qu'il est actif, générer du trafic, puis lire les verdicts des flux en temps réel : ce qui passe (FORWARDED) et ce qui est refusé par les politiques (DROPPED).

Progression0 / 4

Étape 1 · Vérifier que Hubble est actif

Contrôlez l'état de Cilium et de Hubble dans le Terminal : 
kubectl -n kube-system exec ds/cilium -- cilium status | grep -i hubble
Vous devez voir Hubble: Ok. Sinon, activez-le avec cilium hubble enable.

Étape 2 · Générer du trafic autorisé et l'observer

Générez un appel frontend → backend, puis lisez les flux récents : 
BACKEND_IP=$(kubectl get pod -n policy-demo -l app=backend -o jsonpath='{.items[0].status.podIP}')
kubectl exec -n policy-demo deploy/frontend -- curl -s -o /dev/null http://$BACKEND_IP
kubectl -n kube-system exec ds/cilium -- hubble observe --namespace policy-demo --last 5
Le flux frontend → backend apparaît avec le verdict FORWARDED.

Étape 3 · Observer les flux refusés (DROPPED)

Générez un appel non autorisé backend → frontend, puis filtrez les flux refusés : 
FRONTEND_IP=$(kubectl get pod -n policy-demo -l app=frontend -o jsonpath='{.items[0].status.podIP}')
kubectl exec -n policy-demo deploy/backend -- curl -s --connect-timeout 5 http://$FRONTEND_IP || true
kubectl -n kube-system exec ds/cilium -- hubble observe --verdict DROPPED --last 5
Le flux backend → frontend apparaît en DROPPED avec la raison Policy denied.

Étape 4 · Filtrer les flux par label

Ciblez les flux à destination du backend pour ne voir que ce qui le concerne : 
kubectl -n kube-system exec ds/cilium -- hubble observe --namespace policy-demo --to-label app=backend --last 10
Vous combinez ainsi génération de trafic et observabilité pour auditer précisément vos politiques.

Critères de réussite

  • Hubble est actif (Ok).
  • Un flux autorisé est observé (FORWARDED).
  • Un flux refusé est observé (DROPPED).
  • Le filtrage des flux par label fonctionne.

Dépannage

Si aucun flux n'apparaît, activez Hubble puis attendez quelques secondes avant de réobserver : 
cilium hubble enable
kubectl -n kube-system exec ds/cilium -- cilium status | grep -i hubble
