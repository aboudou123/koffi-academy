Leitfaden zur
Backstage

Internal Developer Portal von Grund auf: Deployment, Service Catalog, Software Templates, TechDocs und Keycloak-Authentifizierung.

Fortgeschritten 90–180 min CNCF Node.js Kubernetes
Was Sie lernen werden

Dieser Leitfaden deckt den kompletten Lebenszyklus eines internen Entwicklerportals ab.

Backstage Deployment
Installation on Kubernetes mit Helm, Konfiguration von app-config und erster Start.
Service Catalog
Registrierung von Diensten, Komponenten und Ressourcen im Backstage-Katalog.
Software Templates
Scaffolder-Templates erstellen: automatisierte Projekt-Initialisierung für Entwickler.
TechDocs
Dokumentation as Code mit MkDocs direkt im Portal veröffentlichen.
Keycloak Auth
OIDC-Authentifizierung mit Keycloak konfigurieren und absichern.
Plugins & API
Backstage-Plugins installieren und die Backend-API für eigene Erweiterungen nutzen.
Lernpfad

Folgen Sie den Modulen der Reihe nach oder springen Sie zum gewünschten Thema.

1
Backstage installieren & konfigurieren
3 Schritte · ~45 min
1

Voraussetzungen prüfen

Node.js ≥ 18, Yarn, Docker und ein laufendes Kubernetes-Cluster (minikube oder kubeadm) werden benötigt.

node --version # ≥ 18 yarn --version kubectl cluster-info
2

Backstage App erstellen

Mit dem offiziellen CLI eine neue Backstage-Instanz anlegen und in das Verzeichnis wechseln.

npx @backstage/create-app@latest cd my-backstage-app yarn dev
3

app-config.yaml anpassen

Basis-URL, Datenbankverbindung (PostgreSQL) und GitHub-Integration in app-config.yaml konfigurieren.

2
Service Catalog einrichten
4 Schritte · ~30 min
1

catalog-info.yaml erstellen

Jede Komponente (Service, Bibliothek, Website) wird durch eine catalog-info.yaml beschrieben.

apiVersion: backstage.io/v1alpha1 kind: Component metadata: name: my-service annotations: github.com/project-slug: org/my-service spec: type: service lifecycle: production owner: team-platform
2

GitHub Discovery aktivieren

Backstage automatisch alle Repositories einer Organisation scannen lassen.

3

Beziehungen definieren

DependsOn, ownedBy und partOf setzen, um Abhängigkeiten sichtbar zu machen.

4

Catalog-UI validieren

Im Backstage-Portal überprüfen, ob alle Entities korrekt erkannt und angezeigt werden.

3
Software Templates & Scaffolder
3 Schritte · ~45 min
1

Template-Struktur verstehen

Ein Software Template besteht aus einer template.yaml und einem skeleton/-Verzeichnis mit Vorlagendateien.

2

Node.js-Service-Template erstellen

Formularfelder (Parameter), Aktionen (fetch, publish, catalog:register) und den Skeleton kombinieren.

steps: - id: fetch name: Fetch Template action: fetch:template input: url: ./skeleton - id: publish name: Publish to GitHub action: publish:github - id: register name: Register in Catalog action: catalog:register
3

Template testen

Im Backstage-Portal "Create" öffnen, Template auswählen, Formular ausfüllen und das automatisch erstellte Repository prüfen.

4
Keycloak Authentifizierung
3 Schritte · ~30 min
1

Keycloak-Realm einrichten

Einen Realm, einen Client (OIDC) und Benutzergruppen in Keycloak konfigurieren.

2

Backstage OIDC-Provider konfigurieren

Client-ID und Secret in app-config.yaml eintragen und den Keycloak-Provider aktivieren.

auth: providers: oidc: development: metadataUrl: https://keycloak.example.com/realms/backstage/.well-known/openid-configuration clientId: ${KEYCLOAK_CLIENT_ID} clientSecret: ${KEYCLOAK_CLIENT_SECRET}
3

Sign-In-Resolver implementieren

Benutzer aus dem Keycloak-Token mit Catalog-Entities verknüpfen und Gastmodus deaktivieren.
