Instructions

HTTP API avec API Gateway

Vous créez une HTTP API (API Gateway v2), la câblez à votre Lambda via une intégration proxy, ajoutez les routes GET /items et POST /items, déployez sur un stage prod, et testez les endpoints publics avec curl. La protection JWT viendra au chapitre 3.

Progression0 / 4

Étape 1 · HTTP API & intégration Lambda

Chargez les identifiants sauvegardés, récupérez l'ARN de la Lambda, créez l'HTTP API puis l'intégration proxy. Dans le Terminal : 
source /root/serverless-lab/lab-ids.env
LAMBDA_ARN=$(aws lambda get-function \
  --function-name serverless-lab-api \
  --query 'Configuration.FunctionArn' --output text)
API_ID=$(aws apigatewayv2 create-api \
  --name serverless-lab-api --protocol-type HTTP \
  --query 'ApiId' --output text)
INTEGRATION_ID=$(aws apigatewayv2 create-integration \
  --api-id "$API_ID" --integration-type AWS_PROXY \
  --integration-uri "$LAMBDA_ARN" --payload-format-version 2.0 \
  --query 'IntegrationId' --output text)

Étape 2 · Routes & permission d'invocation

Créez les routes GET /items et POST /items, puis autorisez API Gateway à invoquer la Lambda : 
aws apigatewayv2 create-route --api-id "$API_ID" \
  --route-key "GET /items" --target "integrations/$INTEGRATION_ID"
aws apigatewayv2 create-route --api-id "$API_ID" \
  --route-key "POST /items" --target "integrations/$INTEGRATION_ID"
aws lambda add-permission \
  --function-name serverless-lab-api \
  --statement-id apigateway-invoke \
  --action lambda:InvokeFunction \
  --principal apigateway.amazonaws.com \
  --source-arn "arn:aws:execute-api:us-east-1:${ACCOUNT_ID}:${API_ID}/*"

Étape 3 · Déployer sur le stage prod

Créez le stage prod avec auto-deploy, puis construisez l'URL de l'endpoint : 
aws apigatewayv2 create-stage --api-id "$API_ID" --stage-name prod --auto-deploy
API_URL="https://${API_ID}.execute-api.us-east-1.amazonaws.com/prod"
echo "API URL: $API_URL"

Étape 4 · Tester l'endpoint public

Postez un item, listez-les, puis sauvegardez l'ID et l'URL de l'API pour les chapitres suivants : 
curl -s -X POST "$API_URL/items" \
  -H "Content-Type: application/json" \
  -d '{"itemId": "item-001", "name": "Widget"}' | python3 -m json.tool
curl -s "$API_URL/items" | python3 -m json.tool
echo "export API_ID=$API_ID" >> /root/serverless-lab/lab-ids.env
echo "export API_URL=$API_URL" >> /root/serverless-lab/lab-ids.env
echo "export INTEGRATION_ID=$INTEGRATION_ID" >> /root/serverless-lab/lab-ids.env

Critères de réussite

  • HTTP API et intégration Lambda créées.
  • Routes GET/POST + permission d'invocation.
  • Stage prod déployé, URL obtenue.
  • POST puis GET publics réussis (Widget visible).
