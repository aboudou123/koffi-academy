Objectifs d'apprentissage

  • Préparer les nœuds : swap, modules noyau et paramètres sysctl.
  • Installer et configurer le runtime de conteneurs containerd.
  • Installer kubeadm, kubelet et kubectl à une version épinglée.
  • Initialiser le control plane avec kubeadm init.
  • Configurer l'accès kubectl via le kubeconfig admin.
  • Déployer le plugin réseau Calico (CNI).
  • Joindre des nœuds workers au cluster avec un token.
  • Vérifier l'état des nœuds et des pods système.

Déroulé du laboratoire

  1. Préparer les nœuds

    Désactivez le swap, chargez overlay et br_netfilter, puis réglez les paramètres sysctl réseau.

  2. Installer containerd

    Installez containerd et activez SystemdCgroup = true dans sa configuration.

  3. Installer les binaires Kubernetes

    Installez kubeadm, kubelet et kubectl à une version épinglée, puis figez-les.

  4. Initialiser le control plane

    Lancez kubeadm init avec --pod-network-cidr adapté au CNI choisi.

  5. Configurer kubectl

    Copiez admin.conf dans ~/.kube/config pour piloter le cluster.

  6. Déployer Calico

    Appliquez le manifest Calico et attendez que les pods kube-system soient Ready.

  7. Joindre les workers

    Exécutez kubeadm join avec le token et le hash du certificat sur chaque worker.

  8. Vérifier le cluster

    Contrôlez kubectl get nodes et déployez un nginx de test pour valider le réseau.
