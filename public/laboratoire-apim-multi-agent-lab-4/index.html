Instructions

Dry-run & porte de validation humaine

Vous remplacez le court-circuit --auto-approve par une vraie porte d'approbation : le supervisor publie le plan dans une fixture Slack puis attend un verdict. Vous ajoutez aussi un mode --dry-run qui imprime le plan et sort, sans contacter Slack ni rien appliquer.

Progression0 / 4

Étape 1 · Démarrer la fixture Slack (port 9008)

Inspectez slack_fixture.py dans l'onglet Approval (quatre endpoints : auth.test, chat.postMessage, approvals, healthz), puis démarrez-la : 
bash /root/capstone/fixtures/slack-fixture/run-fixture.sh
curl -sf http://localhost:9008/healthz | jq

Étape 2 · Réécrire ApproveAsync (Slack)

Le script remplace le court-circuit --auto-approve par un vrai flux chat.postMessage + polling, et ajoute un flag --dry-run à Program.cs : 
bash /root/capstone/scripts/patch-supervisor-approval-gate.sh

Confirmez dans Workflow.cs que ApproveAsync lit SLACK_FIXTURE_URL, POSTe le plan sur /api/chat.postMessage et interroge /api/approvals/{plan_id} chaque seconde, jusqu'à 300 secondes.

Étape 3 · Construire & exécuter en dry-run

Compilez, puis lancez avec --dry-run : seul le plan doit s'imprimer, sans contacter Slack ni rien appliquer : 
cd /root/capstone/Supervisor && dotnet build -c Release
dotnet run -c Release -- --dry-run "Publish a new orders API at /orders/v1 with rate limit 100/min"
wc -l /tmp/slack-fixture.log 2>/dev/null || echo 0

Vous ne devez voir que des blocs [plan] — aucun [approve], aucun [apply]. Le log de la fixture Slack reste vide.

Étape 4 · Exécuter avec la porte & approuver

Lancez sans --dry-run ni --auto-approve : le supervisor publie le plan et bloque sur la boucle de polling. Le & final le met en arrière-plan pour approuver depuis le même shell : 
cd /root/capstone/Supervisor && dotnet run -c Release -- "Publish a new orders API at /orders/v1 with rate limit 100/min" &
bash /root/capstone/scripts/approve-latest-plan.sh
tail -40 /tmp/supervisor.log

Vous devez voir la trace complète plan → approve → apply, avec approval_verdict: approve et des blocs [apply] pour GitHub et APIM.

Critères de réussite

  • Fixture Slack démarrée et /healthz OK.
  • ApproveAsync réécrit + flag --dry-run ajouté.
  • Dry-run : plan seul, Slack non contacté.
  • Porte : plan approuvé puis apply exécuté.

Astuce : tapez check dans le terminal pour vérifier votre progression.
