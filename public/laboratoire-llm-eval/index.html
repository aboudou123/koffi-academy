Objectifs d'apprentissage

  • Définir des critères d'évaluation mesurables pour un LLM.
  • Construire un jeu de tests versionné avec réponses de référence.
  • Combiner métriques déterministes et LLM-as-a-judge.
  • Mesurer la faithfulness et la pertinence d'un pipeline RAG.
  • Détecter les régressions entre deux versions de prompt ou de modèle.
  • Intégrer l'évaluation dans un pipeline CI/CD avec seuils.
  • Suivre le coût et la latence par requête d'évaluation.
  • Produire un rapport de qualité lisible par l'équipe.

Environnement virtuel

Sandbox
eval@koffi-lab: ~
# 1 · Définir un jeu de tests (questions + réponses attendues)
$ cat testset.yaml
- question: "Quelle est la capitale de la France ?"
  expected: "Paris"
- question: "Résume l'article en une phrase."
  context:  data/article.md

# 2 · Lancer l'évaluation sur le modèle
$ python eval.py --dataset testset.yaml --model gpt-4o
running 50 cases ............ done
exact_match 0.86   rouge_l 0.71   judge 4.4/5

# 3 · Noter les réponses avec un LLM-juge
$ promptfoo eval -c promptfooconfig.yaml --no-cache
[============] 50/50  pass 47  fail 3
faithfulness 0.93   answer_relevance 0.88

# 4 · Bloquer la CI si le score passe sous le seuil
$ python ci_gate.py --min-score 0.80
quality 0.86 >= 0.80  -> PASS

Déroulé du laboratoire

  1. Préparer l'environnement

    Activez le venv, installez promptfoo, pytest et le SDK du fournisseur, puis configurez votre clé API.

  2. Construire le jeu de tests

    Rédigez un golden set de questions avec réponses de référence et cas limites, versionné dans Git.

  3. Définir les métriques déterministes

    Calculez exact match, ROUGE et BLEU pour les réponses à format attendu.

  4. Ajouter un LLM-as-a-judge

    Notez la justesse et le ton avec un modèle juge et une grille de notation explicite.

  5. Évaluer le pipeline RAG

    Mesurez la faithfulness, la pertinence du contexte et l'absence d'hallucination.

  6. Détecter les régressions

    Comparez deux versions de prompt ou de modèle et identifiez les cas dégradés.

  7. Intégrer dans la CI

    Branchez l'évaluation dans GitHub Actions avec un seuil de score bloquant.

  8. Publier le rapport de qualité

    Générez un rapport HTML avec scores, coûts et latence par cas de test.
