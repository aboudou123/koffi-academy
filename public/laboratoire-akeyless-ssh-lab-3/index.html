Instructions

Émettre & utiliser un certificat

Vous générez une paire de clés SSH locale, demandez à Akeyless un certificat éphémère (5 min) qui lie cette clé à l'utilisateur ubuntu, puis vous connectez au serveur cible — sans jamais ajouter de clé dans authorized_keys.

Les étapes 1 à 3 sont sur l'onglet Terminal (workstation). L'étape 4 vérifie sur l'onglet SSH Target l'absence d'authorized_keys.
Progression0 / 4

Étape 1 · Générer une paire de clés SSH

Terminal · workstation Générez une paire de clés RSA locale ; le certificat sera lié à cette paire. Dans l'onglet Terminal : 
ssh-keygen -t rsa -b 2048 -f /root/.ssh/lab_key -N ""
ls -la /root/.ssh/lab_key*

Étape 2 · Demander un certificat à Akeyless

Terminal · workstation Demandez à Akeyless de signer votre clé publique (TTL 300 s), puis inspectez la fenêtre de validité du certificat : 
akeyless get-ssh-certificate \
  --cert-issuer-name /ssh/lab-issuer \
  --cert-username ubuntu \
  --public-key-file-path /root/.ssh/lab_key.pub \
  --ttl 300 \
  > /root/.ssh/lab_key-cert.pub
ssh-keygen -L -f /root/.ssh/lab_key-cert.pub

Étape 3 · Se connecter au serveur cible

Terminal · workstation Connectez-vous au serveur cible avec votre clé privée et le certificat. Vous devez voir ubuntu et ssh-target : 
ssh -i /root/.ssh/lab_key \
  -o CertificateFile=/root/.ssh/lab_key-cert.pub \
  ubuntu@ssh-target \
  -o StrictHostKeyChecking=no \
  "whoami && hostname"

Étape 4 · Vérifier l'absence d'authorized_keys

SSH Target · serveur cible Sur l'onglet SSH Target, vérifiez qu'aucun authorized_keys n'existe pour ubuntu — le certificat était la seule crédential nécessaire : 
cat /home/ubuntu/.ssh/authorized_keys 2>/dev/null || echo "No authorized_keys file — certificate auth only"
Terminal · workstation Confirmez aussi que le Certificate Issuer est bien configuré : 
akeyless describe-item --name /ssh/lab-issuer

Critères de réussite

  • Paire de clés lab_key / lab_key.pub générée.
  • Certificat éphémère obtenu (TTL 5 min).
  • Connexion réussie (ubuntu @ ssh-target).
  • Aucun authorized_keys utilisé.

Astuce : tapez check dans le terminal pour vérifier votre progression.
