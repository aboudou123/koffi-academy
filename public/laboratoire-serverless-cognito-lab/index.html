Table DynamoDB & fonction Lambda

Vous créez une table DynamoDB Items, un rôle d'exécution IAM pour Lambda, et une fonction Lambda Python qui gère les opérations GET (scan) et POST (put-item). Cette fonction sera exposée par API Gateway au chapitre 2.

Étape 1 · Dossier de travail & table DynamoDB

Définissez la région, créez le dossier de travail, puis la table avec itemId comme clé de partition. Dans le Terminal : 
export AWS_DEFAULT_REGION=us-east-1
mkdir -p /root/serverless-lab
aws dynamodb create-table \
  --table-name Items \
  --attribute-definitions AttributeName=itemId,AttributeType=S \
  --key-schema AttributeName=itemId,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST
aws dynamodb wait table-exists --table-name Items

Étape 2 · Rôle d'exécution Lambda

Récupérez l'ID du compte, créez la trust policy et le rôle, puis attachez une policy donnant les droits DynamoDB CRUD et CloudWatch Logs (les policies sont enregistrées en fichiers dans l'onglet Files) : 
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
cat > /root/serverless-lab/trust-policy.json << 'TRUSTEOF'
{
  "Version": "2012-10-17",
  "Statement": [
    { "Effect": "Allow", "Principal": { "Service": "lambda.amazonaws.com" }, "Action": "sts:AssumeRole" }
  ]
}
TRUSTEOF
ROLE_ARN=$(aws iam create-role \
  --role-name serverless-lab-lambda-role \
  --assume-role-policy-document file:///root/serverless-lab/trust-policy.json \
  --query 'Role.Arn' --output text)
cat > /root/serverless-lab/dynamodb-policy.json << 'POLEOF'
{
  "Version": "2012-10-17",
  "Statement": [
    { "Effect": "Allow", "Action": ["dynamodb:PutItem","dynamodb:GetItem","dynamodb:Scan","dynamodb:DeleteItem"], "Resource": "arn:aws:dynamodb:us-east-1:*:table/Items" },
    { "Effect": "Allow", "Action": ["logs:CreateLogGroup","logs:CreateLogStream","logs:PutLogEvents"], "Resource": "arn:aws:logs:*:*:*" }
  ]
}
POLEOF
aws iam put-role-policy \
  --role-name serverless-lab-lambda-role \
  --policy-name dynamodb-crud-policy \
  --policy-document file:///root/serverless-lab/dynamodb-policy.json
sleep 10

Étape 3 · Fonction Lambda Python

Créez le handler qui lit la méthode HTTP depuis l'event API Gateway et exécute un scan (GET) ou un put-item (POST). Enregistrez lambda_function.py : 
cat > /root/serverless-lab/lambda_function.py << 'PYEOF'
import boto3
import json
import uuid

dynamodb = boto3.resource("dynamodb")
table = dynamodb.Table("Items")

def handler(event, context):
    method = event.get("requestContext", {}).get("http", {}).get("method", "GET")
    if method == "GET":
        result = table.scan()
        return {"statusCode": 200, "headers": {"Content-Type": "application/json"},
                "body": json.dumps(result.get("Items", []))}
    if method == "POST":
        body = json.loads(event.get("body", "{}"))
        item_id = body.get("itemId", str(uuid.uuid4()))
        name = body.get("name", "unnamed")
        table.put_item(Item={"itemId": item_id, "name": name})
        return {"statusCode": 201, "headers": {"Content-Type": "application/json"},
                "body": json.dumps({"itemId": item_id, "name": name})}
    return {"statusCode": 405, "body": "Method Not Allowed"}
PYEOF
Empaquetez et déployez la fonction : 
cd /root/serverless-lab && zip -j lambda.zip lambda_function.py
aws lambda create-function \
  --function-name serverless-lab-api \
  --runtime python3.12 \
  --handler lambda_function.handler \
  --role "$ROLE_ARN" \
  --zip-file fileb:///root/serverless-lab/lambda.zip \
  --timeout 15

Étape 4 · Tester la Lambda & sauvegarder les IDs

Invoquez la Lambda avec un event GET simulé, puis vérifiez la réponse (200, liste vide) et sauvegardez les identifiants pour les chapitres suivants : 
aws lambda invoke \
  --function-name serverless-lab-api \
  --payload '{"requestContext":{"http":{"method":"GET"}}}' \
  --cli-binary-format raw-in-base64-out \
  /root/serverless-lab/response.json
cat /root/serverless-lab/response.json
echo "export AWS_DEFAULT_REGION=us-east-1" > /root/serverless-lab/lab-ids.env
echo "export ACCOUNT_ID=$ACCOUNT_ID" >> /root/serverless-lab/lab-ids.env
echo "export ROLE_ARN=$ROLE_ARN" >> /root/serverless-lab/lab-ids.env
