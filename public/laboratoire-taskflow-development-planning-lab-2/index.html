Etape 1 . Definition du MVP

Creez docs/planning/MVP_FEATURES.md (vision, fonctionnalites, nice-to-have) : 
cat > docs/planning/MVP_FEATURES.md << 'EOF'
# TaskFlow MVP Definition

## Product Vision (One Sentence)
Help small teams organize and track tasks without friction.

## MVP Features (Week 1-6 Deliverables)
- User auth (sign up, login, JWT)
- Create / edit / delete tasks
- Task list with status filters
- Basic dashboard

## Nice-to-Have (Deferred to Post-MVP)
- Real-time collaboration
- Notifications
- Projects and labels
EOF

Etape 2 . Criteres d'acceptation

Definissez des criteres testables dans ACCEPTANCE_CRITERIA.md : 
cat > docs/planning/ACCEPTANCE_CRITERIA.md << 'EOF'
# Acceptance Criteria

## Create a task
GIVEN an authenticated user
WHEN they submit a task with a title
THEN the task appears in their list with status "todo"

## Authentication
GIVEN valid credentials
WHEN the user logs in
THEN they receive a JWT and reach the dashboard
EOF
