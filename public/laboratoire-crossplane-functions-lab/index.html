XRD & installation des fonctions

Étape 1 · Appliquer le XRD

Dans l'onglet Manifests, collez le XRD dans bucket-xrd.yaml (champs buckets array et features), Enregistrez, puis appliquez : 
apiVersion: apiextensions.crossplane.io/v1
kind: CompositeResourceDefinition
metadata:
  name: xbuckets.platform.koffi.academy
spec:
  group: platform.koffi.academy
  names:
    kind: XBucket
    plural: xbuckets
  claimNames:
    kind: BucketClaim
    plural: bucketclaims
  versions:
  - name: v1alpha1
    served: true
    referenceable: true
    schema:
      openAPIV3Schema:
        type: object
        properties:
          spec:
            type: object
            properties:
              environment: { type: string, enum: ["dev","staging","prod"] }
              tier: { type: string, enum: ["basic","standard","premium"] }
              bucketName: { type: string, pattern: "^[a-z][a-z0-9-]*$", maxLength: 20 }
              writeConnectionSecretsToNamespace: { type: string }
              buckets:
                type: array
                items:
                  type: object
                  properties:
                    name: { type: string, pattern: "^[a-z][a-z0-9-]*$", maxLength: 20 }
                    environment: { type: string, enum: ["dev","staging","prod"] }
                    tier: { type: string, enum: ["basic","standard","premium"] }
                  required: [name, environment, tier]
              features:
                type: object
                properties:
                  versioning: { type: boolean, default: false }
                  encryption: { type: boolean, default: false }
            required: [bucketName, environment, tier, writeConnectionSecretsToNamespace]
kubectl apply -f /root/crossplane/manifests/bucket-xrd.yaml
kubectl get xrd

Étape 3 · Composition en mode Pipeline

Collez la composition dans bucket-composition-functions.yaml — un step function-go-templating (logique conditionnelle) puis function-auto-ready — Enregistrez et appliquez : 
apiVersion: apiextensions.crossplane.io/v1
kind: Composition
metadata:
  name: bucket-aws-functions
  labels: { provider: aws, service: bucket, version: functions }
spec:
  compositeTypeRef:
    apiVersion: platform.koffi.academy/v1alpha1
    kind: XBucket
  mode: Pipeline
  pipeline:
  - step: create-bucket-resources
    functionRef: { name: function-go-templating }
    input:
      apiVersion: gotemplating.fn.crossplane.io/v1beta1
      kind: GoTemplate
      source: Inline
      inline:
        template: |
          {{- $b := .observed.composite.resource.spec }}
          {{- $name := printf "koffi-crossplane-%s-%s" $b.bucketName $b.environment }}
          ---
          apiVersion: s3.aws.upbound.io/v1beta1
          kind: Bucket
          metadata:
            name: {{ $name }}
            annotations:
              gotemplating.fn.crossplane.io/composition-resource-name: main-bucket
          spec:
            forProvider:
              region: us-east-1
              tags:
                Environment: {{ $b.environment }}
                Tier: {{ $b.tier }}
                ManagedBy: crossplane-functions
            providerConfigRef: { name: default }
          {{ if or (eq $b.environment "staging") (eq $b.environment "prod") }}
          ---
          apiVersion: s3.aws.upbound.io/v1beta1
          kind: BucketVersioning
          metadata:
            name: {{ $b.bucketName }}-{{ $b.environment }}-versioning
            annotations:
              gotemplating.fn.crossplane.io/composition-resource-name: bucket-versioning
          spec:
            forProvider:
              bucket: {{ $name }}
              region: us-east-1
              versioningConfiguration:
                - status: Enabled
            providerConfigRef: { name: default }
          {{ end }}
          {{ if eq $b.tier "premium" }}
          ---
          apiVersion: s3.aws.upbound.io/v1beta1
          kind: BucketIntelligentTieringConfiguration
          metadata:
            name: {{ $b.bucketName }}-{{ $b.environment }}-tiering
            annotations:
              gotemplating.fn.crossplane.io/composition-resource-name: intelligent-tiering
          spec:
            forProvider:
              bucket: {{ $name }}
              name: entire-bucket
              region: us-east-1
              status: Enabled
            providerConfigRef: { name: default }
          {{ end }}
  - step: auto-ready
    functionRef: { name: function-auto-ready }
kubectl apply -f /root/crossplane/manifests/bucket-composition-functions.yaml
kubectl get composition bucket-aws-functions -o wide

Étape 4 · Tester une claim (dev/basic)

Collez la claim dans test-bucket.yaml (dev/basic → aucune feature conditionnelle), Enregistrez, appliquez et tracez le pipeline : 
apiVersion: platform.koffi.academy/v1alpha1
kind: BucketClaim
metadata:
  name: testbucket
  namespace: backend-team
spec:
  bucketName: testapp
  environment: dev
  tier: basic
  writeConnectionSecretsToNamespace: backend-team
kubectl apply -f /root/crossplane/manifests/test-bucket.yaml
crossplane beta trace bucketclaim testbucket -n backend-team

Critères de réussite

  • XRD xbuckets appliqué au cluster.
  • Trois fonctions installées et HEALTHY.
  • Composition Pipeline créée.
  • Claim dev/basic testée (config minimale).
