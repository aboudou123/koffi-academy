Instructions

Clé CA SSH & Certificate Issuer

Vous créez dans Akeyless une clé d'autorité de certification (CA) RSA — sa clé privée ne quitte jamais Akeyless — exportez sa clé publique, puis définissez un SSH Certificate Issuer et un rôle d'accès. Ce sont les fondations de l'émission de certificats SSH éphémères.

Ce chapitre se déroule entièrement sur l'onglet Terminal (workstation). L'onglet SSH Target servira au chapitre 2.
Progression0 / 4

Étape 1 · Authentifier la CLI & créer la clé CA

Terminal · workstation Configurez la CLI avec votre API Key (depuis la console Akeyless, section Auth Methods), vérifiez l'accès, puis créez une clé RSA qui servira d'autorité de certification SSH : 
akeyless configure --profile default --access-id <YOUR_ACCESS_ID> --access-key <YOUR_ACCESS_KEY>
akeyless list-items
akeyless create-classic-key --name /ssh/lab-ca-key --alg RSA2048
akeyless describe-item --name /ssh/lab-ca-key

Étape 2 · Exporter la clé publique de la CA

Terminal · workstation Exportez la partie publique de la CA. C'est la seule partie qui doit quitter Akeyless — elle sera distribuée aux serveurs pour leur indiquer quelle CA approuver : 
akeyless export-classic-key --name /ssh/lab-ca-key --export-public-key > /root/workspace/akeyless_ca.pub
cat /root/workspace/akeyless_ca.pub

Vous devez voir une clé publique au format ssh-rsa AAAA....

Étape 3 · Créer le SSH Certificate Issuer

Terminal · workstation Créez un Certificate Issuer qui définit les règles d'émission. --allowed-users restreint les comptes accessibles ; --ttl 600 fixe le TTL maximum (10 min) : 
akeyless create-ssh-cert-issuer \
  --name /ssh/lab-issuer \
  --signer-key-name /ssh/lab-ca-key \
  --allowed-users ubuntu \
  --ttl 600

Étape 4 · Créer un rôle d'accès

Terminal · workstation Créez un rôle autorisant la demande de certificats, puis associez votre méthode d'authentification (API Key) à ce rôle : 
akeyless create-role --name /ssh/lab-role
akeyless set-role-rule --role-name /ssh/lab-role --path '/ssh/*' --capability read
akeyless auth-method list
akeyless assoc-role-am --role-name /ssh/lab-role --am-name <YOUR_AUTH_METHOD_NAME>

Critères de réussite

  • CLI authentifiée & clé CA /ssh/lab-ca-key créée.
  • Clé publique de la CA exportée.
  • SSH Certificate Issuer créé (TTL max 600 s).
  • Rôle créé et associé à l'auth method.

Astuce : tapez check dans le terminal pour vérifier votre progression.
