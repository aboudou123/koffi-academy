Étape 2 · Écrire le script de validation

Toujours dans le terminal, collez ce second bloc. Il écrit validate_dataset.py et le rend exécutable avec chmod +x. Le script vérifie les champs obligatoires, les identifiants en double, la longueur des questions et la présence de chaque mot-clé dans la réponse de référence. 
cat > /root/evaluation/datasets/validate_dataset.py << 'EOF'
#!/usr/bin/env python3
"""Structure check for an LLM evaluation set — Koffi Academy."""
import json
import sys
from collections import Counter

REQUIRED = ["id", "question", "reference_answer",
            "expected_keywords", "category", "difficulty"]

def validate(path):
    try:
        with open(path, encoding="utf-8") as fh:
            data = json.load(fh)
    except FileNotFoundError:
        print("FAIL: file not found -> %s" % path)
        return False
    except json.JSONDecodeError as err:
        print("FAIL: invalid JSON -> %s" % err)
        return False

    if not isinstance(data, list) or not data:
        print("FAIL: the dataset must be a non-empty JSON array")
        return False

    problems, seen = [], set()
    for index, row in enumerate(data):
        tag = row.get("id", "row[%d]" % index)

        for field in REQUIRED:
            if field not in row or row[field] in ("", [], None):
                problems.append("%s: missing or empty field '%s'" % (tag, field))

        if tag in seen:
            problems.append("%s: duplicate id" % tag)
        seen.add(tag)

        if len(row.get("question", "")) < 12:
            problems.append("%s: question too short" % tag)

        answer = row.get("reference_answer", "").lower()
        for kw in row.get("expected_keywords", []):
            if kw.lower() not in answer:
                problems.append("%s: keyword '%s' missing from the answer" % (tag, kw))

    if problems:
        print("FAIL: %d issue(s) found" % len(problems))
        for item in problems:
            print("  - " + item)
        return False

    cats = Counter(r["category"] for r in data)
    diffs = Counter(r["difficulty"] for r in data)
    avg_kw = sum(len(r["expected_keywords"]) for r in data) / len(data)
    print("OK: %d entries, %d unique ids" % (len(data), len(seen)))
    print("   categories : %s" % dict(cats))
    print("   difficulty : %s" % dict(diffs))
    print("   avg keywords: %.1f per question" % avg_kw)
    return True

if __name__ == "__main__":
    target = sys.argv[1] if len(sys.argv) > 1 else "/root/evaluation/datasets/kubernetes_eval.json"
    sys.exit(0 if validate(target) else 1)
EOF

chmod +x /root/evaluation/datasets/validate_dataset.py

Étape 3 · Lancer la validation

Activez l'environnement Python (déjà préparé dans /root/evaluation/venv), puis lancez le script sur votre fichier : 
source /root/evaluation/venv/bin/activate
python /root/evaluation/datasets/validate_dataset.py /root/evaluation/datasets/kubernetes_eval.json
Un message OK suivi du nombre d'entrées, des catégories et de la répartition par difficulté confirme que le jeu est prêt. En cas de FAIL, corrigez les lignes signalées et relancez. Astuce : tapez check à tout moment.

Critères de réussite

  • Le fichier kubernetes_eval.json existe et contient un JSON valide.
  • Le script validate_dataset.py est présent et s'exécute sans erreur.
  • Le jeu compte au moins 5 entrées d'évaluation.
  • Chaque entrée porte tous les champs requis et des mots-clés cohérents.

Astuce : tapez check dans le terminal à tout moment pour vérifier votre progression.
