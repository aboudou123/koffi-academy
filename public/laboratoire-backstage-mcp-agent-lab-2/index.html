La boucle ReAct

La boucle ReAct est le cœur agentique que le client déterministe n'avait pas : un prompt produit une séquence raisonnement → outil → résultat → raisonnement. Vous remplissez deux fonctions dans agent.py : strip_fences (3 lignes) et run_agent (~30 lignes), puis vous lancez l'agent.

Étape 1 · Implémenter strip_fences

Dans l'onglet Agent Code, remplacez le corps de strip_fences. Il pèle défensivement les blocs XML <function_calls>, le fence markdown, puis localise l'enveloppe JSON : 
    cleaned = _LLM_XML_RE.sub("", text)
    fence_match = _LLM_FENCE_RE.search(cleaned)
    if fence_match:
        cleaned = fence_match.group(1)
    envelope = _find_json_envelope(cleaned)
    if envelope is not None:
        return envelope.strip()
    return cleaned.strip()

Étape 2 · Implémenter run_agent

Remplacez le corps de run_agent : construisez le client OpenAI avec les variables de l'AI Gateway, bouclez en appelant claude-haiku, enregistrez les tokens, et gérez final_answer / call_tool : 
    client = OpenAI(
        base_url=os.environ["OPENAI_API_BASE"],
        api_key=os.environ["OPENAI_API_KEY"],
    )
    log = TokenLog()
    _ALLOWED = {"catalog.get-catalog-entity", "catalog.query-catalog-entities"}
    raw_tools = mcp_client.list_tools()
    available = [
        {"name": t["name"], "description": (t.get("description") or "").strip().splitlines()[0][:160]}
        for t in raw_tools if t.get("name") in _ALLOWED
    ]
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": (
            f"User prompt: {prompt}\n\nAvailable tools (JSON):\n{json.dumps(available, indent=2)}"
        )},
    ]
    for turn in range(1, max_turns + 1):
        resp = client.chat.completions.create(
            model="claude-haiku", max_tokens=600, messages=messages,
        )
        usage = resp.usage
        log.record(turn, usage.prompt_tokens, usage.completion_tokens)
        raw = resp.choices[0].message.content or ""
        try:
            parsed = json.loads(strip_fences(raw))
        except json.JSONDecodeError:
            return f"agent: failed to parse model output as JSON: {raw}", log
        if "final_answer" in parsed:
            return parsed["final_answer"], log
        if "call_tool" in parsed:
            result = mcp_client.call_tool(parsed["call_tool"], parsed.get("arguments", {}))
            messages.append({"role": "assistant", "content": raw})
            messages.append({"role": "user", "content": f"Tool result (JSON):\n{json.dumps(result, default=str)}"})
            continue
        return f"agent: unrecognized response shape: {raw}", log
    return "agent: turn cap reached", log

Le prompt système dit au modèle d'utiliser catalog.get-catalog-entity avec {kind, namespace, name} — jamais un entityRef.

Étape 3 · Lancer l'agent

Après avoir enregistré agent.py, lancez l'agent sur le prompt canonique dans le Terminal : 
python agent.py 'who owns the payments-api service?'

Critères de réussite

  • strip_fences implémenté (XML + fence + enveloppe).
  • run_agent implémenté (boucle ReAct + TokenLog).
  • Agent répond et affiche son journal de tokens.
