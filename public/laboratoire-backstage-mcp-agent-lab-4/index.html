Rentabiliser : prompt ambigu

Le chapitre 3 a mesuré honnêtement le surcoût. Ici vous le rentabilisez : lancez l'agent sur un prompt ambigu — find the service that owns the /payments path — que le client déterministe ne peut pas traiter, et capturez la trace.

Progression0 / 4

Étape 1 · Lancer le prompt ambigu

Dans le Terminal, lancez l'agent sur le prompt ambigu et tee la sortie : 
python agent.py 'find the service that owns the /payments path' | tee /tmp/agent-task4-output.txt

Étape 2 · Comparer au déterministe

Reproduisez ce que ferait le client déterministe : une simple correspondance de nom (substring) : 
python -c "import json, mcp_client; allc = mcp_client.call_tool('catalog.query-catalog-entities', {'filter': {'kind': 'Component'}}); print('name-only match:', [e['metadata']['name'] for e in allc if 'payments' in e.get('metadata', {}).get('name','')])"

Étape 3 · Le coût du run ambigu

Extrayez le total de tokens du run ambigu (typiquement 30-60% de plus que le prompt simple) : 
grep "Total tokens" /tmp/agent-task4-output.txt

Étape 4 · Vérifier la réponse

Confirmez que l'agent a nommé payments-api (sa description mentionne « Owns the /payments and /refunds HTTP paths ») : 
grep -A 1 "=== Agent answer ===" /tmp/agent-task4-output.txt

Critères de réussite

  • Prompt ambigu exécuté, trace capturée.
  • Limite du client déterministe constatée.
  • Coût du run ambigu extrait.
  • Réponse correcte (payments-api) vérifiée.

Bravo — agent ReAct construit, coût mesuré, et capacité gagnée prouvée !
