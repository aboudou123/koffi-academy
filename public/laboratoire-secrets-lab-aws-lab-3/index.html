Pattern write-only (hors state)

Argument secret_string_wo pour garder le secret hors du state Terraform.

Progression0 / 4

Etape 1 . Ressource write-only

Reecrivez main.tf avec l'argument write-only secret_string_wo (et retirez la data source/output du chapitre 2) : 
cat > /root/terraform/main.tf << 'EOF'
terraform {
  required_providers {
    aws    = { source = "hashicorp/aws",    version = "~> 5.0" }
    random = { source = "hashicorp/random", version = "~> 3.0" }
  }
}
provider "aws" { region = "us-east-1" }

resource "random_password" "db_password" {
  length  = 20
  special = true
}
resource "aws_secretsmanager_secret" "db_password" {
  name = "lab/db_password"
}
resource "aws_secretsmanager_secret_version" "db_password" {
  secret_id                = aws_secretsmanager_secret.db_password.id
  secret_string_wo         = random_password.db_password.result
  secret_string_wo_version = 1
}
EOF

Etape 2 . Appliquer la configuration

Enregistrez et appliquez le plan : 
cd /root/terraform
terraform plan -out=tfplan
terraform apply tfplan

Etape 3 . Inspecter le state (write-only)

Le champ write-only n'est pas persiste dans le state. La valeur en clair a disparu : 
terraform state show aws_secretsmanager_secret_version.db_password

Etape 4 . Confirmer l'absence du secret

Cherchez la valeur dans le fichier de state : elle ne doit plus apparaitre : 
grep -c secret_string_wo terraform.tfstate

Criteres de reussite

  • Ressource reecrite avec secret_string_wo.
  • terraform apply reussi.
  • Valeur absente du state (write-only).
  • Secret toujours present dans AWS.
