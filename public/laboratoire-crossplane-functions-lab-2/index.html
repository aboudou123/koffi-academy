Instructions

Tableaux de buckets

Vous implémentez une logique Go templating avancée pour gérer des tableaux de buckets S3 et des features conditionnelles — la vraie puissance des composition functions : une seule claim, plusieurs buckets, chacun avec sa configuration par environnement et tier.

Étape 1 · Composition avec range

Collez la composition array dans bucket-composition-arrays.yaml — une boucle range gère bucket unique OU tableau — Enregistrez et appliquez : 
apiVersion: apiextensions.crossplane.io/v1
kind: Composition
metadata:
  name: bucket-aws-arrays
  labels: { provider: aws, service: bucket, version: arrays }
spec:
  compositeTypeRef:
    apiVersion: platform.koffi.academy/v1alpha1
    kind: XBucket
  mode: Pipeline
  pipeline:
  - step: create-bucket-resources
    functionRef: { name: function-go-templating }
    input:
      apiVersion: gotemplating.fn.crossplane.io/v1beta1
      kind: GoTemplate
      source: Inline
      inline:
        template: |
          {{- $spec := .observed.composite.resource.spec -}}
          {{- $buckets := list -}}
          {{- if $spec.buckets -}}
            {{- $buckets = $spec.buckets -}}
          {{- else -}}
            {{- $single := dict "name" $spec.bucketName "environment" $spec.environment "tier" $spec.tier -}}
            {{- $buckets = list $single -}}
          {{- end -}}
          {{- range $index, $bucket := $buckets }}
          {{- $name := printf "koffi-crossplane-%s-%s-%d" $bucket.name $bucket.environment $index }}
          ---
          apiVersion: s3.aws.upbound.io/v1beta1
          kind: Bucket
          metadata:
            name: {{ $name }}
            annotations:
              gotemplating.fn.crossplane.io/composition-resource-name: bucket-{{ $index }}
          spec:
            forProvider:
              region: us-east-1
              tags:
                Environment: {{ $bucket.environment }}
                Tier: {{ $bucket.tier }}
                ArrayIndex: "{{ $index }}"
            providerConfigRef: { name: default }
          {{ if or (eq $bucket.environment "staging") (eq $bucket.environment "prod") }}
          ---
          apiVersion: s3.aws.upbound.io/v1beta1
          kind: BucketVersioning
          metadata:
            name: koffi-crossplane-{{ $bucket.name }}-{{ $bucket.environment }}-{{ $index }}-versioning
            annotations:
              gotemplating.fn.crossplane.io/composition-resource-name: versioning-{{ $index }}
          spec:
            forProvider:
              bucketRef: { name: {{ $name }} }
              region: us-east-1
              versioningConfiguration:
                - status: Enabled
            providerConfigRef: { name: default }
          {{ end }}
          {{ if eq $bucket.tier "premium" }}
          ---
          apiVersion: s3.aws.upbound.io/v1beta1
          kind: BucketIntelligentTieringConfiguration
          metadata:
            name: koffi-crossplane-{{ $bucket.name }}-{{ $bucket.environment }}-{{ $index }}-tiering
            annotations:
              gotemplating.fn.crossplane.io/composition-resource-name: tiering-{{ $index }}
          spec:
            forProvider:
              bucketRef: { name: {{ $name }} }
              name: entire-bucket
              region: us-east-1
              status: Enabled
            providerConfigRef: { name: default }
          {{ end }}
          {{- end }}
  - step: auto-ready
    functionRef: { name: function-auto-ready }
kubectl apply -f /root/crossplane/manifests/bucket-composition-arrays.yaml
kubectl get composition

Étape 2 · Tester un bucket unique (staging)

Collez la claim dans single-bucket.yaml (staging/standard → versioning activé), Enregistrez, appliquez et tracez : 
apiVersion: platform.koffi.academy/v1alpha1
kind: BucketClaim
metadata:
  name: single-bucket
  namespace: backend-team
spec:
  bucketName: webapp
  environment: staging
  tier: standard
  writeConnectionSecretsToNamespace: backend-team
  features: { versioning: true, encryption: true }
  compositionRef: { name: bucket-aws-arrays }
kubectl apply -f /root/crossplane/manifests/single-bucket.yaml
crossplane beta trace bucketclaim single-bucket -n backend-team

Étape 3 · Tester un tableau (3 buckets)

Collez la claim multi-buckets dans multi-bucket.yaml — 3 buckets, env/tier différents — Enregistrez, appliquez et listez les buckets : 
apiVersion: platform.koffi.academy/v1alpha1
kind: BucketClaim
metadata:
  name: team-buckets
  namespace: backend-team
spec:
  bucketName: placeholder
  environment: dev
  tier: basic
  writeConnectionSecretsToNamespace: backend-team
  buckets:
    - { name: users,  environment: dev,     tier: basic }
    - { name: orders, environment: dev,     tier: standard }
    - { name: stats,  environment: staging, tier: premium }
  features: { versioning: true, encryption: false }
  compositionRef: { name: bucket-aws-arrays }
kubectl apply -f /root/crossplane/manifests/multi-bucket.yaml
kubectl get bucket.s3.aws.upbound.io
