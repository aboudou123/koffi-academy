Instructions

Configurer le serveur cible

Vous configurez le serveur cible pour faire confiance à la CA Akeyless. C'est la seule modification requise côté serveur : ensuite, tout certificat signé par la CA est accepté sans aucune entrée dans authorized_keys.

La plupart des commandes s'exécutent sur l'onglet SSH Target (en surbrillance ambre). Une étape utilise l'onglet Terminal (workstation) pour copier la clé.
Progression0 / 4

Étape 1 · Vérifier que sshd tourne

SSH Target · serveur cible Sur l'onglet SSH Target, confirmez que le serveur SSH est actif : 
sudo systemctl status sshd

Étape 2 · Copier la clé publique de la CA

Terminal · workstation Sur l'onglet Terminal, affichez la clé publique exportée au chapitre 1 : 
cat /root/workspace/akeyless_ca.pub
SSH Target · serveur cible Puis, sur l'onglet SSH Target, écrivez-la dans le fichier des CA de confiance : 
printf '%s\n' 'PASTE_THE_CA_PUBLIC_KEY_HERE' | sudo tee /etc/ssh/akeyless_ca.pub
cat /etc/ssh/akeyless_ca.pub

Étape 3 · Ajouter TrustedUserCAKeys

SSH Target · serveur cible Ajoutez la directive qui indique à sshd d'approuver les certificats signés par cette CA : 
echo "TrustedUserCAKeys /etc/ssh/akeyless_ca.pub" | sudo tee -a /etc/ssh/sshd_config

Étape 4 · Valider & recharger sshd

SSH Target · serveur cible Validez la syntaxe de la configuration, puis rechargez le démon. Tout client présentant un certificat signé par votre CA pour l'utilisateur ubuntu sera accepté : 
sudo sshd -t && echo "Config OK"
sudo systemctl reload sshd

Critères de réussite

  • sshd actif et en cours d'exécution.
  • Clé publique de la CA écrite sur la cible.
  • Directive TrustedUserCAKeys ajoutée.
  • Config valide et sshd rechargé.

Astuce : tapez check dans le terminal pour vérifier votre progression.
