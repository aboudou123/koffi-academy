Instructions

Supervisor plan → approve → apply

Vous construisez un supervisor (console .NET 8, Microsoft Agent Framework) avec un graphe explicite à trois états : plan → approve → apply. Chaque état possède un seul spécialiste qui appelle un seul serveur MCP. Ce chapitre utilise un court-circuit --auto-approve ; le chapitre 4 le remplace par une vraie validation humaine.

Progression0 / 4

Étape 1 · Échafauder le projet Supervisor

Le script d'échafaudage écrit un projet .NET 8 sous /root/capstone/Supervisor/ avec le jeu de paquets Microsoft Agent Framework 1.6.1 épinglé (il ne lance pas encore le supervisor) : 
bash /root/capstone/scripts/scaffold-supervisor.sh
Ouvrez Program.cs et Workflow.cs dans l'onglet Supervisor : CapstonePlan circule dans les trois états, et chaque état est un petit exécuteur.

Étape 2 · Construire le supervisor

Restaurez les paquets et compilez. Le premier restore tire Microsoft.Agents.AI 1.6.1 et le paquet .Workflows : 
cd /root/capstone/Supervisor && dotnet build -c Release

Étape 3 · Câbler aux pods AKS + AI Gateway

Le supervisor a besoin de trois bundles : l'endpoint Azure OpenAI (ou l'AI Gateway), l'endpoint HTTP de l'APIM MCP (port-forwardé depuis AKS) et le serveur MCP GitHub (lancé via docker run). Le script les câble tous : 
bash /root/capstone/scripts/wire-supervisor.sh

Cela écrit .env avec LLM_API_BASE, LLM_API_KEY, APIM_MCP_URL et GITHUB_MCP_IMAGE, et démarre un port-forward vers l'APIM MCP sur http://localhost:18080/mcp.

Étape 4 · Exécuter en mode auto-approve

Lancez le supervisor avec le prompt de test et le flag --auto-approve (il contourne la porte d'approbation pour clore ce chapitre sur un aller-retour plan → apply propre) : 
cd /root/capstone/Supervisor && dotnet run -c Release -- --auto-approve "Publish a new orders API at /orders/v1 with rate limit 100/min"
tail -80 /tmp/supervisor.log

Vous devez voir trois blocs : [plan] avec la proposition, [approve] avec auto-approved, et [apply] avec le commit GitHub simulé et la création APIM.

Critères de réussite

  • Projet Supervisor échafaudé.
  • Build en Release réussi.
  • Câblage .env + port-forward APIM MCP.
  • Run auto-approve : plan → approve → apply.

Astuce : tapez check dans le terminal pour vérifier votre progression.
