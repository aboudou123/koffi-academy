Objectifs d'apprentissage

  • Naviguer dans la console AWS et comprendre régions et zones.
  • Configurer et utiliser l'AWS CLI en toute sécurité.
  • Explorer les services clés : S3, EC2, IAM, VPC.
  • Créer et détruire des ressources sans risque dans le bac à sable.
  • Provisionner de l'infrastructure avec Terraform.
  • Comprendre les bonnes pratiques de coûts et de sécurité.

Environnement virtuel

Sandbox
aws@koffi-playground: ~
# 1 · Vérifier votre identité dans le bac à sable
$ aws sts get-caller-identity
{ "Account": "1234-sandbox", "Arn": "arn:aws:iam::…:user/lab" }

# 2 · Explorer S3 et EC2
$ aws s3 ls
$ aws ec2 describe-regions --query 'Regions[].RegionName'
[ "eu-central-1", "us-east-1", … ]

# 3 · Créer un bucket de test (puis le supprimer)
$ aws s3 mb s3://koffi-lab-demo-01
$ aws s3 rb s3://koffi-lab-demo-01

# 4 · Provisionner avec Terraform
$ terraform init && terraform plan
$ terraform apply -auto-approve

# 5 · Tout nettoyer pour éviter les coûts
$ terraform destroy -auto-approve
Destroy complete! Resources: 3 destroyed.

Déroulé du laboratoire

  1. Ouvrir le bac à sable

    Lancez l'environnement : console AWS + terminal avec l'aws-cli déjà configurée.

  2. Vérifier votre identité

    Exécutez aws sts get-caller-identity pour confirmer votre accès au compte bac à sable.

  3. Explorer les services

    Parcourez S3, EC2, IAM et VPC depuis la console et la CLI, sans risque.

  4. Provisionner avec Terraform

    Initialisez un projet, lancez terraform plan puis apply pour créer des ressources.

  5. Nettoyer

    Détruisez tout avec terraform destroy et videz les buckets pour éviter tout coût.
