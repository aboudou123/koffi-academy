Architecture systeme

Architecture trois tiers de TaskFlow : composants, interactions et ADR.

Progression0 / 4

Etape 1 . Document d'architecture systeme

Creez docs/architecture/SYSTEM-ARCHITECTURE.md decrivant l'architecture trois tiers : 
cat > docs/architecture/SYSTEM-ARCHITECTURE.md << 'EOF'
# TaskFlow System Architecture

## Overview
TaskFlow is a three-tier web application: a Vue.js frontend, a FastAPI
backend and a PostgreSQL database.

## Components
### 1. Frontend (Vue.js)
Render task lists and dashboards, handle interactions, call the API.
### 2. Backend API (FastAPI)
Expose REST endpoints, business logic, auth, data validation.
### 3. Database (PostgreSQL)
Persist users, tasks and projects with relational integrity.

## Component Interaction
User -> Frontend -> REST/HTTPS -> Backend API -> SQL -> Database.

## Non-Functional Requirements
- Scalability: stateless API, horizontal scaling
- Reliability: health checks, retries
- Security: JWT auth, TLS, least privilege
EOF

Etape 2 . Architecture Decision Record

Documentez le choix du backend dans un ADR : 
cat > docs/architecture/ADR-001-CHOOSE-FASTAPI.md << 'EOF'
# ADR-001: Choose FastAPI for the backend

## Status
Accepted

## Context
We need a Python backend with async support and automatic API docs.

## Decision
Use FastAPI for the REST API.

## Consequences
- High performance (ASGI), OpenAPI docs out of the box
- Type-safe request/response models via Pydantic
EOF
