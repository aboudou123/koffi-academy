Instructions

Bridges VLAN-aware

Avec votre conception en place, vous configurez des bridges VLAN-aware sur les nœuds simulés. Une seule interface physique porte ainsi le trafic de nombreux réseaux virtuels, chaque VM étant taguée sur un VLAN précis — c'est le mécanisme d'isolation tenant de votre Tâche 1.

Progression0 / 4

Étape 1 · Lister les interfaces du nœud 1

Observez que seul le bridge à plat vmbr0 est configuré (eth2/eth3 inutilisés) : 
./scripts/pve-net-sim.sh show pve-node1

Étape 2 · Revoir l'exemple

Notez les deux lignes clés : bridge-vlan-aware yes et bridge-vids 100-200 : 
cat configs/vlan-bridge-example.conf

Étape 3 · Créer & appliquer le bridge

Dans l'onglet Lab Files, créez vmbr0-vlan.conf (management 10.0.1.0/24 + VLAN-aware), Enregistrez, puis appliquez : 
auto eth0
iface eth0 inet manual

auto vmbr0
iface vmbr0 inet static
    address 10.0.1.11/24
    gateway 10.0.1.1
    bridge-ports eth0
    bridge-stp off
    bridge-fd 0
    bridge-vlan-aware yes
    bridge-vids 100-200
./scripts/pve-net-sim.sh apply-bridge vmbr0 /root/proxmox-net-lab/configs/vmbr0-vlan.conf

Étape 4 · Vérifier le tagging VLAN

Confirmez que les VLANs 100–200 sont actifs sur vmbr0 : 
./scripts/pve-net-sim.sh verify-vlans pve-node1

Critères de réussite

  • État initial du nœud 1 observé.
  • Exemple VLAN-aware compris.
  • vmbr0 VLAN-aware créé et appliqué.
  • VLANs 100-200 actifs vérifiés.
