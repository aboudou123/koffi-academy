Objectifs d'apprentissage

  • Distinguer root, service et batch tokens et leurs cas d'usage.
  • Créer des tokens avec policies, TTL et max TTL explicite.
  • Inspecter un token via son accessor sans jamais l'exposer.
  • Renouveler et prolonger la durée de vie d'un token.
  • Révoquer un token et la cascade de ses enfants.
  • Configurer des token roles et des periodic tokens.
  • Appliquer le moindre privilège aux identités machine.
  • Tracer les opérations de tokens via les audit devices.

Environnement virtuel

Sandbox
vault@koffi-lab: ~
# 1 · Créer un token de service avec policy et TTL
$ vault token create -policy=apps -ttl=1h -explicit-max-ttl=4h
token                hvs.CAESIJ...redacted
token_accessor       8sh2mK0pQ...
token_duration       1h    token_policies  [default apps]

# 2 · Inspecter via l'accessor (le token n'est jamais affiché)
$ vault token lookup -accessor 8sh2mK0pQ...
ttl   59m48s    num_uses   0    orphan   false

# 3 · Renouveler le token (+30 min)
$ vault token renew -increment=30m hvs.CAESIJ...
token_duration       30m

# 4 · Révoquer le token et toute sa descendance
$ vault token revoke hvs.CAESIJ...
Success! Revoked token (if it existed)

Déroulé du laboratoire

  1. Démarrer Vault et s'authentifier

    Lancez Vault en mode dev, exportez VAULT_ADDR et connectez-vous avec le root token fourni.

  2. Écrire une policy applicative

    Définissez une policy apps au moindre privilège, limitée aux chemins de secrets nécessaires.

  3. Créer un token de service

    Générez un token rattaché à la policy avec un TTL d'1 h et un max TTL explicite de 4 h.

  4. Inspecter via l'accessor

    Utilisez l'accessor pour auditer le token (TTL restant, policies, orphan) sans jamais l'exposer.

  5. Renouveler et observer l'expiration

    Prolongez la durée de vie, puis laissez un token court expirer pour comprendre le lease.

  6. Révoquer en cascade

    Révoquez un token parent et vérifiez la révocation automatique de ses enfants ; testez -mode=orphan.

  7. Token roles et periodic tokens

    Créez un token role réutilisable et un periodic token pour les charges de longue durée.

  8. Activer l'audit

    Branchez un audit device fichier et retrouvez chaque opération de token dans le journal.
