Instructions

Expiration TTL & piste d'audit

Vous observez l'expiration TTL de votre certificat, confirmez qu'un certificat expiré est rejeté, ré-émettez-en un frais, et passez en revue les données d'analyse Akeyless. La piste d'audit complète — impossible avec authorized_keys — enregistre chaque certificat émis.

Ce chapitre se déroule sur l'onglet Terminal (workstation). Le certificat du chapitre 3 est désormais expiré.
Progression0 / 4

Étape 1 · Inspecter la fenêtre de validité

Terminal · workstation Examinez la période de validité du certificat actuel — il était valide 5 minutes depuis son émission au chapitre 3 : 
ssh-keygen -L -f /root/.ssh/lab_key-cert.pub | grep -E "Type|Signing CA|Valid"

Étape 2 · Tenter d'utiliser le certificat expiré

Terminal · workstation Tentez de vous connecter : vous devez obtenir une erreur d'expiration. Le certificat est cryptographiquement invalide après son TTL, sans aucun moyen de le prolonger sans Akeyless : 
ssh -i /root/.ssh/lab_key \
  -o CertificateFile=/root/.ssh/lab_key-cert.pub \
  ubuntu@ssh-target \
  -o StrictHostKeyChecking=no

Étape 3 · Ré-émettre un certificat frais

Terminal · workstation Demandez un nouveau certificat pour regagner l'accès, puis reconnectez-vous : 
akeyless get-ssh-certificate \
  --cert-issuer-name /ssh/lab-issuer \
  --cert-username ubuntu \
  --public-key-file-path /root/.ssh/lab_key.pub \
  --ttl 300 \
  > /root/.ssh/lab_key-cert.pub
ssh -i /root/.ssh/lab_key \
  -o CertificateFile=/root/.ssh/lab_key-cert.pub \
  ubuntu@ssh-target \
  -o StrictHostKeyChecking=no \
  "echo 'Re-connected successfully'"

Étape 4 · Examiner les données d'analyse

Terminal · workstation Affichez les statistiques d'usage agrégées de votre compte. Chaque demande de certificat est aussi journalisée dans l'Audit Log de la console Akeyless — une traçabilité impossible avec authorized_keys : 
akeyless get-analytics-data
akeyless describe-item --name /ssh/lab-issuer

Critères de réussite

  • Fenêtre de validité du certificat inspectée.
  • Certificat expiré rejeté (Certificate invalid: expired).
  • Certificat ré-émis et reconnexion réussie.
  • Données d'analyse / audit consultées.

Astuce : tapez check dans le terminal pour vérifier votre progression.

Bravo — vous avez remplacé les authorized_keys par des certificats SSH éphémères, audités, émis par Akeyless !
