Objectifs d'apprentissage

  • Concevoir un plan d'adressage CIDR cohérent, sans chevauchement.
  • Créer un VPC et activer la résolution DNS et les hostnames.
  • Répartir des sous-réseaux publics et privés sur deux zones de disponibilité.
  • Attacher une Internet Gateway et router le trafic sortant public.
  • Déployer une NAT Gateway pour la sortie des sous-réseaux privés.
  • Écrire et associer des tables de routage publiques et privées.
  • Vérifier la connectivité depuis une instance publique et privée.
  • Détruire proprement les ressources pour éviter tout coût résiduel.

Environnement virtuel

Sandbox
aws@koffi-lab: ~
# 1 · Créer le VPC (CIDR /16) et activer le DNS
$ aws ec2 create-vpc --cidr-block 10.0.0.0/16 --query Vpc.VpcId --output text
vpc-0a1b2c3d4e5f6789
$ aws ec2 modify-vpc-attribute --vpc-id vpc-0a1b… --enable-dns-hostnames

# 2 · Créer les sous-réseaux publics sur deux AZ
$ aws ec2 create-subnet --vpc-id vpc-0a1b… --cidr-block 10.0.1.0/24 --availability-zone eu-central-1a
$ aws ec2 create-subnet --vpc-id vpc-0a1b… --cidr-block 10.0.2.0/24 --availability-zone eu-central-1b

# 3 · Internet Gateway et route par défaut publique
$ aws ec2 create-internet-gateway --query InternetGateway.InternetGatewayId --output text
igw-0f9e8d7c6b5a
$ aws ec2 create-route --route-table-id rtb-pub --destination-cidr-block 0.0.0.0/0 --gateway-id igw-0f9e8d…

# 4 · NAT Gateway pour la sortie des sous-réseaux privés
$ aws ec2 allocate-address --domain vpc --query AllocationId --output text
eipalloc-07c3f2a1
$ aws ec2 create-nat-gateway --subnet-id subnet-pub1 --allocation-id eipalloc-07c…

# 5 · Vérifier la topologie créée
$ aws ec2 describe-subnets --filters Name=vpc-id,Values=vpc-0a1b… --query 'Subnets[].CidrBlock'
[ "10.0.1.0/24", "10.0.2.0/24", "10.0.11.0/24", "10.0.12.0/24" ]

Déroulé du laboratoire

  1. Planifier l'adressage CIDR

    Découpez le bloc 10.0.0.0/16 en sous-réseaux /24 par zone, publics et privés, sans chevauchement.

  2. Créer le VPC et activer le DNS

    Créez le VPC puis activez enableDnsSupport et enableDnsHostnames.

  3. Créer les sous-réseaux

    Créez deux sous-réseaux publics et deux privés répartis sur eu-central-1a et eu-central-1b.

  4. Attacher une Internet Gateway

    Créez l'IGW et attachez-la au VPC pour permettre l'accès Internet.

  5. Table de routage publique

    Créez une route table, ajoutez la route 0.0.0.0/0 → igw et associez les sous-réseaux publics.

  6. Déployer une NAT Gateway

    Allouez une Elastic IP et placez une NAT Gateway dans un sous-réseau public.

  7. Routage privé

    Créez une route table privée avec 0.0.0.0/0 → nat et associez les sous-réseaux privés.

  8. Vérifier et nettoyer

    Testez la connectivité, puis détruisez NAT, EIP, sous-réseaux, IGW et VPC pour éviter les coûts.
